# Contributing to Pothole Detection System

Thank you for your interest in contributing to the Pothole Detection System! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature or bug fix
4. Make your changes
5. Test your changes thoroughly
6. Submit a pull request

## Development Setup

1. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

2. **Run the application locally**
   ```bash
   python app.py
   ```

3. **Test your changes**
   - Test the web interface at `http://localhost:5000`
   - Verify detection functionality works
   - Check all pages load correctly

## Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and small

## Submitting Changes

1. **Create a descriptive commit message**
   ```
   Add feature: Real-time detection confidence adjustment
   
   - Added slider to adjust confidence threshold
   - Updated detection.js to handle dynamic thresholds
   - Modified backend API to accept confidence parameter
   ```

2. **Push to your fork and submit a pull request**

3. **Describe your changes**
   - What problem does this solve?
   - How did you test it?
   - Any breaking changes?

## Reporting Issues

When reporting issues, please include:
- Operating system and version
- Python version
- Browser (if web-related)
- Steps to reproduce
- Expected vs actual behavior
- Error messages or logs

## Feature Requests

We welcome feature requests! Please:
- Check if the feature already exists
- Describe the use case
- Explain why it would be valuable
- Consider implementation complexity

## Areas for Contribution

- **Bug fixes**: Check the issues page
- **Documentation**: Improve README, add tutorials
- **Testing**: Add unit tests, integration tests
- **Performance**: Optimize detection speed
- **UI/UX**: Improve web interface design
- **Mobile**: Mobile-responsive improvements
- **Accessibility**: Make the app more accessible

## Questions?

Feel free to open an issue for questions or join discussions in existing issues.
